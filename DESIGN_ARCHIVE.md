# 设计存档：极简高尔夫原型

更新时间：2026-04-22

## 项目定位

本项目是一个 Godot 4 的极简 3D 高尔夫单洞原型。它不是完整产品，也不是长期架构练习，当前阶段只验证几件核心事情：

- 程序化生成器能否稳定生成可读、可玩的单洞地形。
- 球的击打、滚动、飞起和落地是否有清晰手感。
- 镜头、地形和视觉参照是否能让玩家读懂空间。
- 入洞后自动生成下一洞的连续节奏是否成立。

项目最初的方向是“极简禅意、低 UI、低机制、低视觉噪音”。后续视觉方向调整为在这个基础上加入“克制梦幻赛博”：允许网格线、青紫粉色彩、按坡度和朝向着色，但仍然避免强烈霓虹、复杂后处理和装饰物堆叠。

## 核心约束

当前原型必须保持小工程：

- 不做菜单系统。
- 不做多球杆、风、升级、商店、成就或长期进度。
- 不做复杂 HUD。
- 不做装饰物生成系统。
- 不做 ECS、事件总线、服务定位器、插件系统、通用内容数据库。
- 不为未来假想需求建立 manager 层、工厂层或重型架构。
- 优先快速可玩验证，而不是可扩展性。

玩法循环保持为：

```text
生成一洞 -> 击球 -> 计杆 -> 入洞 -> 下一洞
```

## 已完成内容

### 常驻规则

已创建 `AGENTS.md`，作为后续 Codex 工作前必须阅读的中文项目约束。它记录了项目目标、禁止事项、生成哲学、视觉方向、UI 规则、镜头规则、物理规则和工程规模限制。

### 最小 Godot 项目

当前项目已经具备最小 Godot 4 结构：

- `project.godot`：配置主场景和基础输入。
- `scenes/main.tscn`：唯一主可玩场景。
- `scripts/main.gd`：主游戏循环、输入、球、镜头、UI、自动测试。
- `scripts/hole_generator.gd`：程序化球洞、地形、碰撞、颜色、网格线和验证。

这个结构刻意保持小规模，目前没有额外场景、菜单、素材管线或独立系统层。

### 单洞可玩验证

已实现 seed 驱动的单洞生成与基本玩法：

- 启动后自动生成一洞。
- 玩家可以瞄准、蓄力并击球。
- 每次击球会统计杆数。
- 球进入洞杯范围后自动生成下一洞。
- 仍只验证当前单洞循环，没有扩展到完整产品。

### 自动 AI 三洞测试

已加入命令行自动测试模式：

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . --scene res://scenes/main.tscn -- --autoplay-test
```

自动 AI 可以读取球、洞杯、距离、速度和当前地形信息来估算击球，但最终仍通过正常击球流程完成，不传送球、不直接设置入洞状态、不扩大玩家入洞规则。

最近一次通过记录：

- 第 1 洞：3 杆，约 16.15 秒。
- 第 2 洞：3 杆，约 16.73 秒。
- 第 3 洞：3 杆，约 15.37 秒。
- 总计：3 洞完成，约 50.45 秒。

### Push / Hit 双模式

为了解决球几乎只贴地滚动、缺少“击球感”的问题，已加入两种最小击球模式：

- `PUSH`：主要水平冲量，适合贴地滚动和近距离推杆。
- `HIT`：水平冲量加明显向上分量，让球能离地再落下。

输入仍保持简单：

- `A/D` 或方向键：瞄准。
- `Space` 或鼠标左键：蓄力和击球。
- `Tab`：切换 `PUSH` / `HIT`。
- `R`：重置当前洞。

屏幕只额外显示一个很小的当前模式文本，不增加复杂 UI。

### 高度可读性

为了解决地面太平、球飞起后高度难以判断的问题，已加入两个低噪音参照：

- 地形颜色中加入极淡的高度和层次变化。
- 球正下方加入半透明地面影子，球越高影子越淡、越小。

这些参照不是 HUD，而是空间阅读辅助，目的是让 Hit 模式下的离地高度更容易判断。

### 克制赛博地块地形

视觉方向进一步调整为“克制梦幻赛博”，并把地形生成从连续高度场推进为真实的程序化地块地形：

- 仍然先生成空间骨架，再生成地形。
- 沿发球点到洞杯的路径生成不同长度、不同宽度、不同斜度的地块。
- 每个地块有自己的基础高度、前后坡度、侧向坡度和过渡。
- 发球区和洞杯区仍强制平缓，避免破坏可玩性。
- 最终仍采样成单个地形 mesh 和碰撞体，保持工程简单。

地形颜色现在根据坡度和朝向变化：

- 坡度越大，颜色更偏亮或偏冷。
- 朝向在青、紫、粉之间轻微偏移。
- 颜色服务于读地形，不做随机花色。

同时加入克制的赛博网格线：

- 网格线提供高度和空间参照。
- 使用低噪音的青色或淡紫色。
- 不引入复杂 shader 或材质系统。

## 当前实现摘要

### `scripts/main.gd`

负责当前原型的大部分运行逻辑：

- 创建和重置球。
- 管理当前 seed、杆数、入洞和下一洞。
- 处理玩家输入、瞄准、蓄力、释放击球。
- 管理 `PUSH` / `HIT` 模式。
- 控制相机跟随和基础灯光氛围。
- 显示极简杆数和击球模式文本。
- 更新球影子。
- 在 `--autoplay-test` 模式下运行自动 AI 三洞测试。

这个文件有意没有拆成多个 manager，以避免第一轮原型过度工程化。

### `scripts/hole_generator.gd`

负责程序化球洞生成：

- 根据 seed 创建直视型单洞参数。
- 先建立 tee 到 hole 的空间骨架。
- 再沿骨架生成地块布局。
- 根据地块平面、边缘 falloff 和少量噪声采样地形高度。
- 抚平发球区和洞杯区。
- 生成 terrain mesh、vertex color、碰撞和赛博网格线。
- 执行最小验证，拒绝过陡、过窄或洞杯区不稳定的结果。
- 返回发球点、洞杯点、当前参数和安全球道信息，供主循环和自动测试使用。

## 后续迭代提醒

下一步仍应保持小范围迭代。优先观察和调整：

- 地块坡度是否足够有读数，但不过分破坏可玩性。
- `HIT` 模式的飞行高度、落点和节奏是否舒服。
- 赛博网格线是否帮助读地形，而不是变成视觉噪音。
- 自动三洞测试是否仍能作为物理和生成回归。
- 是否需要微调球的阻尼、摩擦、上抛力度和地形坡度。

暂时不建议扩展：

- 新洞型全集。
- 多球杆系统。
- 菜单和关卡选择。
- 复杂 AI。
- 成就、进度、商店或长期系统。
- 大规模脚本拆分或架构重构。

当前最重要的方向仍是：让一个小小的程序化单洞循环，在地形、球感、镜头和视觉阅读上变得更清楚、更好玩。
